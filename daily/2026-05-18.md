## 📝 Learning Notes

### Cryptography Core Concepts

**Hash** — 不是加密，是承诺
- 任意输入 → 固定长度输出（fingerprint）
- 改一个字，输出完全不同
- 用途：交易哈希定位交易、区块哈希定位区块、数据承诺（先公开哈希再公开原文验证）

**Public Key** — 可公开，推导地址、验证签名
- 和私钥成对出现
- 地址 = 公钥的短标识
- 地址可以公开，但不能仅凭"看起来像"就信任

**Private Key** — 控制权本身
- 生成签名，证明账户控制
- 泄漏 = 资产直接被转走
- 铁律：不截图、不上云、不贴网页、不给 AI、不放进代码仓库、不用主钱包测陌生 dApp

**Signature** — 不是"登录弹窗"
- 对具体消息或交易内容的授权证明
- 用户必须看懂签名内容（domain, chain ID, contract address, amount, expiry）
- AI Agent 场景：签名必须有权限边界，不能让模型随意触发高风险授权

**Merkle Tree** — 高效验证
- 逐层哈希 → 一个 root
- Merkle proof 证明"我的数据在这批数据里"，不用下载全部
- 常见于空投名单、轻客户端、状态证明

### Connection to AI × Web3

> AI Agent 如果要解释交易、判断授权、生成链上操作或帮助用户管理权限，**必须理解签名和密钥的边界**。模型可以解释"这笔授权在批准 token 额度"，但不能替用户保管私钥。AI 输出本身也可能需要被签名/哈希/记录，作为审计证据。
