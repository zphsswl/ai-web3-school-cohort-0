### Handbook Key Concepts (Agentic Commerce)

- **Agentic Commerce 第一性原理**: Agent 可以替用户发起商业动作，但不能替用户承担无限责任 → 购买意图、预算、验收标准和争议规则必须是结构化协议
- **Payment Intent**: 不是「帮我买最合适的服务」，而是「买什么、最多花多少、接受什么结果」的结构化表达 → 你的 `TransactionIntent` 已有 input/output/amount/budget/slippage 字段
- **Budget Control 分层**: 任务预算 / 服务预算 / 时间预算 / 风险预算 / 失败预算 → 你的 `budget.py` 已实现前 3 层
- **Proof of Task Completion**: 不是单一证明，而是一组证据（API response + tx hash + event logs + acceptance records）→ CLI `history` + UserOp hash 是最基础实现
- **On-chain Receipt**: payer/provider/task id/payment amount/tx hash/output hash → 你的 UserOp 执行日志天然包含这些字段
