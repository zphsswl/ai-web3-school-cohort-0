## 📖 Reading Path

### 🟢 Minimum Path (≈30 min)

> 目标：理解机器支付的核心哲学——把"付款意图"和"实际结算"拆开，每一步都有凭证

- [ ] 读 **第一性原理** (5 min) — 三大原则：
  1. 预算先于执行：没有预算边界就没有安全自动支付
  2. 报价必须可比较：价格、币种、有效期、服务范围、退款条件
  3. 收据必须可验证：付给谁、为什么付、交付了什么
- [ ] 读 **Stablecoin Payment** (初级, 5 min) — 稳定币是机器支付的天然计价单位；需处理 chain id / token address / decimals / allowance / gas；approve 是高风险动作不可与普通付款混用
- [ ] 读 **Budget** (初级, 5 min) — 多层预算模型：全局预算 → 任务预算 → 单次调用上限 → 服务方上限 → 紧急停止；防诱导：Agent 即使被诱导反复调用也只能消耗小范围额度
- [ ] **最小实践** (10 min) — 设计 Agent 购买 API 的支付流程：
  > 用户授权：今天最多 3 USDC
  > API 返回 quote：一次调用 0.1 USDC，5 分钟内有效
  > Agent 检查预算和服务方身份
  > 钱包或支付工具完成付款
  > API 返回结果和 receipt
  > 系统记录 quote、payment intent、交易哈希、结果、剩余预算
- [ ] 在下方 ## 📝 Learning Notes 写 3-5 条关键收获

### 🟡 Recommended Path (≈1 hour)

> 目标：深入理解机器支付的每个环节，掌握 Quote / Payment Intent / x402 的差异

- [ ] 完成 🟢 Minimum Path 全部内容 (30 min)
- [ ] 读 **Quote** (中级, 6 min) — 可执行报价的 7 要素：服务内容、价格、币种、收款地址、有效期、交付条件、退款条件、quote id；过期 quote 的危险性；需要签名/验证来源
- [ ] 读 **Payment Intent** (中级, 6 min) — 与 Quote 的关系：Intent 表达"用户授权这类付款"，Quote 是服务方的具体报价；Payment Intent 至少包含用户目标、服务方、最大金额、币种、链、过期时间、quote 引用、是否允许自动重试、是否需要人工确认
- [ ] 读 **x402** (中级, 5 min) — HTTP 402 变成互联网原生支付流程；Agent 可像处理 401 登录一样处理 402 付款；但不解决服务质量/退款/争议
- [ ] **扩展练习** (10 min) — 对比 x402 和 MPP 两种支付协议：
  - x402 适合什么场景？（API 调用、按次付费）
  - MPP 适合什么场景？（服务发现→报价→支付→收据全流程）
  - 为什么高价值服务还需要 escrow 和 dispute 机制？
