  - 流程：读输入 → 展示解析结果 → 展示 Policy 检查 → 展示 Budget 检查 → 展示 UserOperation
  - 不需要交互 — 先走通「一句话 → 完整输出」
  - 目标：3 条命令都能跑通（正常 swap / 超限额被拒 / 无效 token 被拒）

### 🟡 Recommended Path (1 hour)

- [ ] **精读 [Agent Workflow](https://aiweb3.school/zh/handbook/bridge/agent-workflow/) — 3 个关键节点** (15 min)
  - **Human-in-the-loop**: CLI 的确认步骤对应分层确认模型。小额（≤$10）自动确认，中额（$10-$100）展示计划+要求输入 'y'，大额（>$100）展示红色警告
  - **Trace**: CLI 的终端输出 = Agent 的 trace。每一行输出都是可追溯的决策记录
  - **State Machine**: CLI 的状态流转：`input → parsed → policy_checked → budget_checked → (pending_confirmation) → userop_built → done | rejected`
- [ ] **参考 [Agent Wallet](https://aiweb3.school/zh/handbook/bridge/agent-wallet/) — Session Key + Guard 展示** (10 min)
  - Session Key 的 5 维约束要在 CLI 里可见：用户应该看到「当前 Session Key 何时过期、还剩多少额度、能调哪些合约」
  - Guard：被拒绝的原因要清楚展示（不是 "rejected"，而是 "滑点 5% > 最大滑点 2%"）
- [ ] **实现完整 CLI — REPL + 单命令双模式** (30 min)
  - REPL 模式：`python cli.py` 进入交互环境，支持 `help` / `wallet status` / `wallet session` / `tx` / `history` / `exit` 命令
  - 单命令模式：`python cli.py --cmd "swap 50 USDC to ETH"`
  - 输出美化：用 emoji 和颜色区分信息/警告/错误；展示交易计划用表格格式
  - 确认流：$10+ 交易需要输入 'y' 确认
- [ ] **集成测试** (5 min)
  - `python cli.py --cmd "swap 50 USDC to ETH"` → 正常通过
  - `python cli.py --cmd "swap 5000 USDC to ETH"` → Policy 拒绝 + 清晰理由
  - `python cli.py --cmd "swap 50 USDC to MALICIOUS_COIN"` → 合约不在白名单

### Handbook Key Concepts (for CLI design)

- **Human-in-the-loop 分层**: "只读分析自动执行 / 交易草稿自动生成 / 小额白名单 session key 执行 / 高风险人工确认 / 超出 policy 直接拒绝"。CLI 在每个层级暂停并展示清晰信息
- **Trace = 终端输出**: Agent Workflow 强调「没有 trace 就不知道问题在哪」。CLI 的终端输出天然就是 trace — 每一行都记录做了什么、为什么
- **Agent Wallet Guard**: 「Guard 是确定性拦截层」— CLI 里 Guard 的拒绝信息必须说清楚 "什么规则被违反了，当前值是什么，限制值是多少"
