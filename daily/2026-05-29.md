> 目标：理解 Agent Wallet 的核心哲学——不是给 AI 一个钱包，而是给它一个受控的行动空间

- [ ] 读 **第一性原理** (5 min) — 三条核心原则：
  1. Agent 不直接控制主资产 → 高风险动作交回给人确认
  2. 钱包不只是签名按钮，而是权限系统 → 时间/金额/合约/方法/资产/接收方/撤销条件
  3. 自动化必须绑定撤销能力 → 用户随时能看到 Agent 拿了什么权限、做过什么、还能做什么、在哪关掉
- [ ] 读 **Smart Account** (中级, 8 min) — 理解合约账户如何承载规则：谁能操作、哪些需确认、白名单、日限额、自动失效、暂停

- [ ] 读 **Human Check** (初级, 5 min) — 风险分层确认：低风险自动→中等风险模拟后确认→高风险明确展示影响→超 policy 直接拒绝

## 💡 Core Takeaway

**Agent Wallet ≠ 给 AI 一个钱包私钥。Agent Wallet = 给 AI 一个可验证、可限制、可撤销的行动空间。**

区分四个层次 — 这也是判断一个 AI × Web3 项目是否靠谱的核心指标：

| 层级 | 做什么 | 谁控制 |
|------|--------|--------|
| 建议层 | Agent 生成方案，不执行 | 用户完全控制 |
| 草稿层 | Agent 生成交易草稿，用户签名 | 用户确认后执行 |
| 受限执行层 | Agent 在 policy 内自动执行 | 系统规则控制 |
| 全权层 | Agent 完全控制资产 | ❌ 不应该存在 |

> *Good Agent Wallet design finds the middle path: let the Agent execute, but only within boundaries the user pre-approved. The key is not "how to sign" but "what can be signed, by whom, under what conditions, and how to revoke."*
