**Handbook link**: [链感知上下文 Chain-aware Context →](https://aiweb3.school/zh/handbook/bridge/chain-aware-context/) · ~5 min read · ~1632 字 · 7 个知识节点

**🎉 Phase 2 begins!** 从今天起，我们从"认识 Web3 组件"转向"让 AI Agent 正确使用 Web3"。Chain-aware Context 是整个 Bridge 的**输入层**——如果 Agent 连自己在哪条链、和哪个合约交互都不知道，后面的 Tool Use、Workflow、Agent Wallet 都建立在不可靠基础上。

**Core takeaway**: 普通 AI 的上下文来自文档、聊天历史、数据库。AI × Web3 多了一层：**链上状态持续变化，且直接关联资产和权限**。Chain-aware Context 的核心是把链上事实 (chain id, contract address, ABI, tx history, citation) 变成模型可读、可引用、可验证的结构化上下文。

## 📖 Reading Path

### 🟢 Minimum Path (30 min)

- [ ] Read [链感知上下文 Chain-aware Context](https://aiweb3.school/zh/handbook/bridge/chain-aware-context/)
    - 第一性原理：
        1. 模型不能凭语言记忆判断链上事实 → 必须从工具和索引层读取
        2. 链上状态有时间性 → 余额/授权随区块变化
        3. 上下文要带来源 → contract address, block number, tx hash, explorer link
        4. 区分事实和解释 → 工具返回事实，模型负责解释，别把猜测当事实
    - 7 个知识节点速览：
        - **On-chain Data** (初级)：RPC/区块浏览器/索引器读取，至少带 chain id + block number + contract + method + result + time
        - **Contract Docs** (初级)：ABI 只有函数签名，文档补充业务语义，但文档也可能过期 → 链上验证
        - **ABI / Event** (中级)：ABI 编码调用/解码返回值，Event 是业务日志 (Transfer/Swap/Deposit)
        - **Transaction History** (中级)：判断用户是否授权过、策略是否执行过、是否和高风险合约交互过
        - **Explorer Context** (初级)：区块浏览器提供可检查入口，给 explorer link 比"交易成功"可靠
        - **Indexing Context** (中级)：面向产品的查询，"用户最近 30 天 DeFi 操作""池子 TVL 变化"，带时间戳和同步状态
        - **Citation** (初级)：无 citation 的链上解释 = 观点，带 citation = 可验证可追责
- [ ] **✍️ 最小实践**：为一笔真实交易构建上下文包
    - 去 [Etherscan](https://etherscan.io/) 随便找一笔公开交易
    - 收集：chain id, block number, from, to, method, value, token transfers, logs
    - 找到合约 ABI 或 verified source
    - 写一段模型可读的上下文，每个关键结论附上 tx hash 或 explorer link
    - 标出：哪些是链上事实 ✅，哪些是你的解释 💭

### 核心概念

**Chain-aware Context 的四个层次**：

| 层次 | 内容 | 来源 |
|------|------|------|
| **网络层** | chain id, block number, RPC provider | RPC / 网络配置 |
| **合约层** | contract address, ABI, verified source, docs | 区块浏览器 / 文档 |
| **状态层** | balance, allowance, tx history, event logs | RPC / 索引器 |
| **验证层** | citation (tx hash, explorer link, block number) | 区块浏览器 |

**为什么这层是"输入层"？**

没有 Chain-aware Context → Agent 只能用模型记忆猜链上状态 → 错误判断 → 错误交易。
有了 Chain-aware Context → Agent 基于可验证的事实做决策 → 每次判断可以追溯 → 用户和系统可以验证。
