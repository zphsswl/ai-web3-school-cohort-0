**Handbook**: https://aiweb3.school/zh/handbook/bridge/ai-privacy/

**核心问题**: 用户数据、钱包身份、链上行为、私有记忆、API key 和模型上下文之间，哪些可以共享？哪些必须隔离？哪些只应该在本地处理？**Web3 数据公开 + AI 扩大数据面 = 链上公开身份和 AI 私有上下文被拼接的风险。**

**第一性原理**: 模型只应该看到完成任务所需的最小数据。不要因为"模型可能有帮助"就把钱包历史、私聊、API key 全塞进上下文。

**8 个知识节点**:

| # | 节点 | 难度 | 一句话 |
|---|------|------|--------|
| 1 | **Data Boundary** | 🟢 初级 | 数据在设备、后端、模型、链上、第三方工具之间如何流动 — 画成数据流图而非隐私政策长文 |
| 2 | **Local AI** | 🟡 中级 | 敏感推理放本地，先筛选脱敏再送强模型 — "本地提取3笔相关交易"而非"上传全部历史" |
| 3 | **Private Memory** | 🟡 中级 | Agent 长期记忆分层管理：短期任务/长期偏好/敏感身份/钱包记录，可查看可删除 |
| 4 | **Secret Management** | 🔴 高级 | 私钥/API key/JWT 绝不进 prompt/日志/analytics；即使用户粘贴也应拒绝处理 |
| 5 | **Minimal Disclosure** | 🟢 初级 | 只暴露最少必要信息 — 证明余额够付 10 USDC 不必暴露全部持仓 |
| 6 | **Encrypted Data** | 🟡 中级 | 加密保护存储/传输，但模型使用时仍需解密 — 关键在组合 access control + TEE + 最小上下文 |
| 7 | **User Consent** | 🟢 初级 | 细粒度同意（连接钱包 ≠ 分析全部历史），可撤销，说清哪些能删哪些不能删 |

### 核心概念 (Key Concepts)

1. **Data Boundary 是第一道防线**
   - 不是写隐私政策，而是画数据流图：每项数据在哪？流向谁？进模型还是进工具？
   - Agent 工具之间数据不应默认互通：浏览器看到的 ≠ 钱包看到的 ≠ 模型看到的
   - 边界要在代码层 enforce，不是靠 prompt 约束

2. **Secret Management 是硬底线**
   - 私钥/助记词/API key 绝不能进 prompt、日志、analytics
   - 即使用户主动粘贴，系统应识别并拒绝
   - 工具调用的 secret 通过受控服务注入，模型永远接触不到明文

3. **Privacy × Security 的关系**
   - Security 防外部攻击，Privacy 防内部误用和过度收集
   - 昨天的 AI Security → 模型犯错不能变成资产损失
   - 今天的 AI Privacy → 没有隐私设计，Agent 越懂用户泄漏后伤害越大
   - 两者共用同一基础设施：Data Boundary、Permission Isolation、Audit Log
