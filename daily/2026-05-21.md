Core takeaway: Account Abstraction liberates "how to verify, who pays gas, which permissions auto-execute" from the fixed EOA model — turning wallets into programmable account systems. The core isn't "free gas," it's expanding account control from a single private key to programmable rules. This is **the foundation** for AI agents to execute on-chain with constrained, auditable, revocable permissions.

### ERC-4337 Flow Summary
```
User/App → creates UserOperation
  → Smart Account verifies (signature, nonce, balance, policy)
  → Bundler collects & simulates & submits
  → EntryPoint calls account to execute
  → Paymaster optionally sponsors gas
```

### Key Permission Shift
- **EOA**: has private key → can do everything
- **Smart Account**: meets conditions → can do specific things
- **Session Key**: within scope (time, contract, method, amount) → auto-execute

### AI x Web3 Relevance
Account Abstraction is NOT about making AI "freer" — it's about wrapping AI freedom in rules:
- What can it call? → method whitelist
- How much? → amount limit
- How long? → expiry
- Who can revoke? → user/guardian
- Where's the log? → on-chain events
- What if it fails? → revert + notify
