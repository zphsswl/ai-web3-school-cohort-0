Core takeaway: Smart contracts are on-chain programs that make rules, assets, and state publicly verifiable. Learning them isn't about writing complex protocols immediately — it's about building judgment: what belongs on-chain, how state changes work, and why testing & auditing can't be skipped.

### Key Concepts Map

| Concept | What it is | Why it matters for AI × Web3 |
|---------|-----------|------------------------------|
| **Solidity** | EVM 合约语言，storage/msg.sender/event | Agent 需要理解合约的读写边界 |
| **EVM** | 合约执行环境，决定 gas/隔离/部署 | Agent 调用需要预估 gas 和处理 revert |
| **ABI** | 机器可读接口描述 | Agent 通过 ABI 理解如何编码调用 |
| **Event** | 可索引链上日志 | Agent 通过事件追踪历史状态变化 |
| **Upgrade** | Proxy/治理/多签升级机制 | Agent 需要知道合约可能被修改 |

### Call Flow

```
用户点击 → 前端读地址/网络 → ABI 编码调用数据 → 钱包展示确认 →
签名广播 → RPC 节点 → 验证者打包进区块 → EVM 执行/更新状态/emit event →
前端监听交易回执 → 索引器读 event 更新数据层
```
