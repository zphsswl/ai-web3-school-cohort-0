**Phase 1: Web3 Foundations · 索引 (Indexing)**

> ⚠️ WCB Learning page uses CSR (Next.js), cannot be scraped. Topic derived from [learning-plan.md](https://github.com/zphsswl/ai-web3-school-cohort-0/blob/master/learning-plan.md) Phase 1 schedule.

**Handbook link**: [索引 Indexing →](https://aiweb3.school/zh/handbook/web3/indexing/) · ~4 min read · ~1330 字 · 4 knowledge nodes

Core takeaway: 链上数据是公开的，但不等于好用。Indexing 把区块、交易、事件和合约状态整理成产品、分析工具和 AI Agent 能快速查询的结构化数据。**链上是事实来源 (source of truth)，索引层是可用数据层 (usable data layer)。** RPC 不是数据库 — 你需要 Event Indexing、Subgraph、Data Pipeline 来构建面向问题的查询模型。

## 📤 Check-in Draft

> **Day 8 · 索引 Indexing · Phase 1 Web3 Foundations**
>
> 今天学的是链上索引层 (Indexing)——把公开但不一定好用的链上数据，变成产品、分析工具和 AI Agent 能快速查询的结构化数据。
>
> **3 个关键收获**：
> 1. **RPC 不是数据库**：eth_call 和 eth_getLogs 能读链状态，但产品查询（用户历史、排行榜、风控）需要专门的索引层
> 2. **Event 是索引入口**：合约 event 是索引器构建状态的主要信号——设计 event 时要考虑后续查询需求（indexed 参数、是否包含关键地址、能否还原业务状态）
> 3. **AI Agent 的上下文来自索引**：交易历史、用户仓位、协议状态、风险信号都不能每次从原始区块搜索——好的索引层提供结构化、带来源、带时间戳、可回溯的数据
>
> **实践完成**：设计了事件索引方案 (event indexing design)，考虑了 reorg/重复事件/合约升级处理 + AI Agent 数据消费需求
>
