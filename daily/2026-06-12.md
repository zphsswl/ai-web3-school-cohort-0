  ```python
  @dataclass
  class BudgetLimits:
      per_tx_usdc: float        # 单笔上限
      per_day_usdc: float       # 每日上限
      max_failures: int          # 连续失败次数上限
      per_service_usdc: float   # 单个服务上限

  @dataclass
  class BudgetTracker:
      daily_spent_usdc: float = 0.0
      daily_txs: int = 0
      consecutive_failures: int = 0
      last_reset_day: str = ""  # "2026-06-12"

  class BudgetController:
      def __init__(self, limits: BudgetLimits):
          self.limits = limits
          self.tracker = BudgetTracker()

      def check(self, amount_usdc: float, service: str) -> tuple[bool, str]:
          """Return (allowed, reason). Layer: per_tx → daily → service."""
          if amount_usdc > self.limits.per_tx_usdc:
              return False, f"exceeds per-tx limit ({self.limits.per_tx_usdc} USDC)"
          if self.tracker.daily_spent_usdc + amount_usdc > self.limits.per_day_usdc:
              return False, "exceeds daily limit"
          # ... service check, failure cap
          return True, ""

      def record_failure(self): ...
      def reset_daily(self): ...
  ```

  ```python
  class WalletCore:
      def __init__(self, policy: PermissionPolicy, budget: BudgetController):
          self.policy = policy
          self.budget = budget

      def can_execute(self, intent: TransactionIntent) -> CheckResult:
          """Permission check → Budget check → combined result"""
          perm_result = self.policy.check(intent)
          if not perm_result.allowed:
              return perm_result
          budget_result = self.budget.check(intent.amount, intent.target_contract)
          return budget_result
  ```

## 📝 Learning Notes

<!-- Your insights from today's reading and building -->

### Machine Payment → AI-native Wallet 映射

1. **Budget > Intent**: 意图告诉你 Agent 想做什么，预算告诉你 Agent 能花多少 — 两者都是权限系统的必选项
2. **Quote + Payment Intent = 交易前验证**: 报价必须可比较（价格/币种/有效期/退款条件），Payment Intent 必须绑定原始授权（谁/什么/多少钱/何时到期）
3. **订阅是隐含危险**: Agent 订阅数据源时，静默续费比单次超支更难察觉 — 必须强制每月上限 + 续费通知 + 随时可撤
4. **Micropayment 先算经济账**: 每次调用 0.001 USDC 的 API 不值得单独上链 → 批量结算或链下账本

### 实现决策

- **v0.1 Budget Controller 先实现三层**: per_tx / per_day / failure_cap — 这是最小安全边界
- **v0.2 再加**: per_service / per_task — 需要服务方注册表后才能做
- **稳定币支付**: USDC on Ethereum 作为默认计价/结算币种，后续支持跨链
- **BudgetController 不与 ERC-4337 耦合**: 先纯 Python 实现，v0.2 再与 Smart Account policy 对接
