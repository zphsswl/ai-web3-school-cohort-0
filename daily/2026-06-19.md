### Machine Payment Key Concepts

- **第一性原理**: 支付能力必须和 Agent Wallet、Policy、Settlement、Receipt 连接。一个可用链路：用户授权预算 → Agent 获取 quote → 系统检查 policy → 付款 → 服务交付 → receipt
- **预算先于执行**: 没有预算边界，就没有安全自动支付
- **Stablecoin Payment**: 稳定币为机器支付提供计价单位，但要处理 chain id、token address、decimals、allowance、gas 和跨链可用性。Approve 本身是高风动作，不能和普通付款混在一起
- **Budget 陷阱**: 只设总预算不设频率和服务方范围 → Agent 可能把全部预算花在一个异常报价上
- **Quote 验证**: 服务方返回的 quote 必须签名、有 quote id、绑定收款地址，否则后续 dispute 无法证明 Agent 接受了什么条件
- **x402**: 让工具可以像处理 401 Login 一样处理 402 Payment — 先读付款要求 → 检查预算 → 付款 → 重放请求
