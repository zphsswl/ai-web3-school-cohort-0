```
Key Concepts:
- Oracle ≠ "real-world API" — 它是链上合约信任的数据提交+验证机制
- Data source = trust boundary — 谁提供、谁聚合、谁能更新，必须清晰
- Latency = risk — 旧价格可能导致错误清算
- 4 个知识节点：Price Feed → Data Feed → Oracle Risk → AI Oracle

Three Oracle Models:
1. Push-based (Chainlink): 链下节点定期推送价格到链上合约
2. Pull-based (Pyth): 用户自行拉取签名价格数据并提交到链上
3. Optimistic (UMA): 提交数据后设挑战期，有争议则进入仲裁

AI Oracle 的关键问题（与普通 Price Feed 对比）：
- 输入数据是否可追溯？模型版本 + prompt 是否记录？
- 输出是否可复核？是否允许挑战/仲裁？
- 错误输出会造成什么链上后果？
```
