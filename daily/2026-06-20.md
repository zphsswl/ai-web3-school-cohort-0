## 📖 Reading Path

### 🟢 Minimum Path (30 min) — 今天必做

- [ ] **README 升级 — 5 段落核心版** (30 min)
  - **Section 1: Overview** (5 min) — 一句话 + 问题陈述 + 方案总结（中英双语，~150 字）
    - EN: "AI-native Wallet v0.1 lets users describe DeFi goals in natural language. The AI parses intent into structured transactions, while a deterministic permission system (Policy → Budget → Session Key) enforces hard constraints before execution."
    - CN: "AI-native Wallet v0.1 让用户用自然语言描述 DeFi 操作意图，AI 解析生成结构化交易，确定性权限系统（策略 → 预算 → 会话密钥）在链上执行前强制校验。"
  - **Section 2: Architecture** (7 min) — ASCII 流程图 + 6 组件职责表
    ```
    User (NL Intent)
          ↓
    CLI / Chat Interface
          ↓
    Intent Parser (NL → Structured Intent)  ← 🧠 LLM
          ↓
    Permission Policy (Rules Check)         ← 🛡️ Deterministic
          ↓
    Budget Controller (Limit Enforcement)   ← 💰 Deterministic
          ↓
    Session Key Manager (Signing)           ← 🔑 Minimal Permission
          ↓
    Smart Account (ERC-4337 UserOp)         ← ⛓️ On-chain
    ```
    职责表：每个组件标注「做什么 ✅」和「不做什么 ❌」
  - **Section 3: Security Model** (8 min) — 4 层安全 + Human-in-the-loop 分层
    - Layer 1: Parser — 翻译不执行（LLM 只生成草案，不触碰资产）
    - Layer 2: Policy — 规则拦截（允许的资产/合约/时间段白名单）
    - Layer 3: Budget — 额度限制（日/周/任务/单次 四维上限）
    - Layer 4: Session Key — 最小权限（限时、限额度、限制具体操作）
    - Human-in-the-loop: <$10 自动 / $10-$100 确认 / $100+ 警告 / 越界拒绝
  - **Section 4: Demo Guide** (5 min) — 4 步演示流程
    - Step 1: 正常 swap — `python cli.py "swap 50 USDC to ETH"`
    - Step 2: 超限被拒 — swap 超过日预算，显示拒绝理由
    - Step 3: 错误合约 — 尝试 transfer 给不在白名单的合约
    - Step 4: 查看状态 — `python cli.py --status` 显示 budget 剩余
  - **Section 5: Setup** (5 min) — 依赖安装 + 环境变量 + 运行命令
    ```bash
    pip install web3.py eth-account python-dotenv
    cp .env.example .env  # fill RPC_URL, LLM_API_KEY
    python cli.py "swap 50 USDC to ETH"
    ```
