### 🟢 Minimum Path (30 min)

- [ ] Read [安全 Security](https://aiweb3.school/zh/handbook/web3/security/) — 重点：第一性原理 + 全部 5 个知识节点概览
  - 第一性原理：链上系统默认在对抗环境里，任何可调用路径都要按攻击面看
  - **Reentrancy** (中级)：Checks-Effects-Interactions 模式、reentrancy guard
  - **Access Control** (中级)：不只问"有没有 onlyOwner"，更问 owner 是谁、有没有 timelock、权限变更是否发 event
  - **Audit** (中级)：审计不是安全保证书，读 audit report 至少看 5 个关键点
  - **Simulation** (中级)：交易执行前预演——调哪个合约、转出什么、gas 多少、是否 revert
  - **Monitoring** (高级)：大额转账、权限变更、合约升级、TVL 异常、Agent 工具触发

```
Security for AI Agents — 关键设计原则：

1. 模型可以建议，不能直接执行 → policy 限制权限
2. 交易前必须 simulation → Tenderly / local fork
3. 高风险动作必须 human check → 大额转账、合约部署、权限变更
4. 上下文必须带来源 → chain id、合约地址、交易哈希、explorer 链接
5. 执行后持续 monitoring → 异常告警 + 响应流程

AI Agent 安全 ≠ 合约安全 + AI 安全，而是两者交叉：
模型可能被 prompt injection → 生成危险交易
工具边界不清 → Agent 调用不该调用的合约
上下文陈旧 → 基于过时状态做决策
```
